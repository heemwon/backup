FAQ

  • 전체(51)

    • Virtual Server

      Virtual Server의 사용료는 어떤 기준으로 과금됩니까?

      VM은 CPU/Memory 크기, 약정 여부 (무약정/1년/3년), OS 종류 (오픈소스, RHEL, Windows)에 따라 요금이 변경 되며, 해당 요금 기준으로 생성한 VM의 사용 시간에 따라 과금됩니다.
    • VM Auto-Scaling

      어떤 기준으로 자원 규모가 조정됩니까?

      VM Auto-Scaling의 실행 조건은 크게 2가지 기준으로 나눌수 있습니다. 첫번째는 자원 사용량 입니다. 사용자는 CPU/메모리 등의 임계치/평균값 등을 지정하여, 자원 생성/삭제의 기준을 정의할 수 있습니다. 두번째는 일정입니다. 사용자는 희망하는 스케쥴을 사전 등록하여, 해당 시점에 자원의 자동 생성/삭제를 설정할 수 있습니다.
    • Bare Metal Server

      Virtual Server와 다른 Bare Metal Server의 제약 조건은 무엇입니까?

      단독 자원을 제공하기 때문에 일부 제공 되지 않는 기능들이 있으며, 지속적으로 개선을 위해 검토 중 입니다.
      - Security Group 설정, Custom Image 생성, Init script 입력 등 기능 미제공
      - 한개의 프로젝트 내 Bare Metal Server는 최대 20개까지 생성 가능
    • Block Storage

      블록스토리지 볼륨 생성시, 최대 할당량은 얼마인가요 ?

      기본적으로 서버의 OS영역은 100GB를 제공하며, 이후 외장 블록 스토리지를 통해 160TB 까지 온라인 중에 할당 가능합니다.
    • File Storage

      파일스토리지 볼륨 생성시, 최대 할당량은 얼마인가요?

      파일스토리지의 볼륨을 생성 시, 최대 100TB 까지 할당할 수 있습니다.
    • Object Storage

      과금은 어떻게 되나요?

      버킷 내 사용량을 측정하여 일단위로 계산되어 과금됩니다.
    • Backup

      백업으로 제공하는 대상은 어떤게 있나요?

      VM 이미지 스냅샷, NAS 이미지 스냅샷, DB 백업(Online) 을 제공합니다.
    • DB Service

      DB Service는 무엇인가요?

      DB Service는 웹 기반 콘솔을 통해 관계형, 비정형 및 데이터 분석 관련 DBMS에 대해 데이터베이스 생성과 설정을 자동화된 구성과 환경으로 제공합니다. 중요 데이터의 안정적인 관리를 위해 VM 환경에서 동작하며, VPC 기반으로 단일 인스턴스나 이중화 구성이 가능하고, 백업/복구, Parameter 관리, Audit , Minor Patch, 모니터링 등 데이터베이스의 생명주기 내 업무를 자동화 기능으로 제공합니다.
    • Kafka

      Kafka를 시작하려면 어떻게 해야 하나요?

      사용자는 SCP Kubernetes Engine을 사용해야 합니다. 만약 그렇지 않은 경우 SCP Kubernetes Engine 사용신청을 통해 사용가능한 워크노드(일반 VM)가 신청합니다. SCP Kubernetes Engine에서 프로비저닝한 워커노드(일반 및 GPU VM)를 Kafka의 자원으로 이용하게 됩니다.
    • API Gateway

      API 등록 시, Endpoint와 연결이 되었는지 확인이 가능한가요?

      네 가능합니다.
      삼성SDS 클라우드 콘솔에 로그인하여 API 상품 화면으로 접속하시어, Service 생성 후 좌측에 [API 관리 > API 등록]에서 'Connection Test' 클릭 시, 확인 가능합니다.
    • Mail/SMS/Push

      API형태로 제공되는 Mail/SMS/Push 서비스 이용 방법이 어떻게 되나요?

      삼성SDS 클라우드 콘솔에 로그인 한 후, 우측 [모든 상품 → Application Service → Mail/SMS/Push]를 통해 상품신청을 합니다. 이 후, 신청된 각 상품별로 [상품목록 → Mail/SMS/Push] 클릭, [사용자 가이드] 조회를 통해 가이드내 API들을 직접 커스텀하여 사용 가능합니다.
    • Kubernetes Engine

      SCP Kubernetes Engine(SKE)를 사용해야 하는 이유는 무엇입니까?

      SCP Kubernetes Engine(SKE) 없이 컨테이너 환경을 구성하고 싶다면, 컨트롤플레인과 워커노드 구성을 위한 VM생성부터 Kuberentes 설치 등 Kubernetes 클러스터 구성을 직접 수행해야 합니다.

      SCP Kubernetes Engine(SKE)를 사용하면 SCP 콘솔에서 간단하게 워커 노드를 프로비저닝할 수 있으며, SCP가 보장하는 가용성의 컨트롤 플레인을 제공받을 수 있습니다. 이를 통해 Kubernetes를 실행하는 데 따른 운영 부담을 현저히 줄이고 컨테이너 환경 구성을 위한 노력을 애플리케이션 구축에 활용할 수 있습니다.

      SCP는 고가용성 보장을 위해 비정상적인 컨트롤 플레인 노드를 감지하고 교체하며 컨트롤 플레인에 대한 패치를 제공합니다. 또한 다양한 SCP의 상품들과 통합하여 애플리케이션에 확장성과 보안을 강화할 수 있습니다.
    • Kubernetes Apps

      Kubernetes Apps는 무엇인가요?

      Kubernetes Apps는 Web/WAS를 비롯한 App Infrastructure 등 다양한 컨테이너 이미지를 Kubernetes Engine으로 생성한 클러스터에 배포하는 기능을 제공하는 서비스입니다.
      다수의 현장 운영 경험을 기반으로 최적화 된 설정의 컨테이너 이미지를 제공하기 때문에 손쉬운 Apps 배포가 가능합니다.
    • VPC

      기존에 사설망(고객망)에서 사용하던 비공인 IP를 클라우드 환경에서도 그대로 사용할 수 있나요?

      고객망에서 사용하던 비공인IP 도 사용 가능합니다만, 마이그레이션 상황(순차적 이관 등)에 따라 기존 사용하던 비공인IP를 그대로 사용하지 못할 수도 있습니다. 그리고 기존 사설망(고객망)과 연동을 위해서는 Direct Connect 또는 VPN 상품을 통해 연결해야 합니다.
    • Security Group

      Security Group과 VPC의 방화벽의 차이점은 무엇인가요?

      Networking의 Security Group은 VM을 위한 방화벽입니다. Security Group에서 작성한 방화벽 Rule은 Virtual Server, Auto-Scaling, DB, Load Balancer 등에서 사용할 수 있습니다.
      VPC 상품 내 방화벽 탭에서 설정하는 룰은 VPC Gateway에서 동작합니다.
    • Load Balancer

      Load Balancer에서 생성할 수 있는 서비스는 어떤 것이 있나요?

      L4 TCP, L7 HTTP 그리고 L7 HTTPS를 제공합니다.
      HTTP/HTTPS의 경우 URL 패턴에 따라 서비스를 전달할 서버그룹을 다르게 지정할 수 있습니다.
      HTTP와 HTTPS 선택 시 URL 처리 입력이 활성화 되고, HTTPS 입력 시 SSL인증서를 필수로 지정해야 합니다.
    • DNS

      이미 보유 중인 도메인은 등록할 수 없는가요?

      신규 도메인 신청만 가능하며, 기존 고객이 보유 중인 도메인 등록은 아직 안됩니다.
      ※ 신규 도메인은 1년 단위로 신청되며, 도메인등로 건당 사용료(자동연장) 별도 청구됩니다.
    • GSLB

      GSLB는 언제 사용하나요?

      GSLB는 DNS 기반으로 지역별 트래픽 증가시 트래픽을 분산하고 특정 서버 장애시 Fail over하여 서비스가 지속될 수 있도록 하는 상품으로, DC(Data Cente)r에 문제가 생길 경우에 다른 지역DC로 연결하여 시스템 가용성을 높여줍니다.
    • VPN

      호환되는 VPN 장비는 어떤 것이 있나요?

      시스코, 팔로알토, 체크포인트, 주니퍼 등이며 IPsec표준을 통해 연동합니다.
      VPN 표준을 준수했다면 연동 시 문제는 크게 없을 것입니다.
      연동 후 안정화 기간을 두고 점검하길 권고드립니다.
    • DDoS 대응

      WAF를 신청하면 DDoS 대응은 자동으로 신청되나요?

      DDoS 대응 상품은 WAF 상품을 이용하는 고객에 한하여 제공됩니다. (Basic 등급에 한함) 다만, DDoS 대응 상품을 이용하기 위해서는 WAF 신청 이후 별도로 신청을 해주셔야 합니다.
    • IDS

      IDS Agent 설치 및 삭제는 누가 하는 것인가요?

      VM에 IDS Agent 설치 및 삭제는 고객이 수행합니다. 서비스 신청 시 E-mail을 통해 관제센터에서 매뉴얼을 제공해 드립니다.
    • WAF

      탐지 모드에서 공격을 시도한 IP 차단이 되나요?

      DDoS 대응 상품은 WAF 상품을 이용하는 고객에 한하여 제공됩니다. (Basic 등급에 한함) 다만, DDoS 대응 상품을 이용하기 위해서는 WAF 신청 이후 별도로 신청을 해주셔야 합니다.
    • 웹 취약점 점검

      웹 취약점 점검 이후 재진단이 하고 싶습니다.

      웹 취약점 점검 완료 후 발견된 취약점에 대해 이행검수를 제공합니다. 취약점에 대해 검수 요청을 해주시면 수동 점검을 수행한 후 결과 Report를 제공해 드립니다. 이행검수 제공기간은 취약점 점검 완료 후 30일 내이며 총 2회까지 요청하실 수 있습니다.
    • 인증서 관리

      인증서 관리는 어떤 상품인가요?

      SSL/TLS 인증서를 등록하여 SDS Cloud를 통해 제공되는 웹사이트/어플리케이션을 보호할 수 있습니다. 또한 인증서 조회, 유효기간 만료 알림 설정 등 통합인증서 관리 기능을 제공합니다.
    • Kubeflow

      SCP Kubeflow를 사용해야 하는 이유는 무엇입니까?

      SCP Kubeflow 없이 ML 환경을 구성하고 싶다면, Kuberntes 컨트롤플레인과 워커노드 구성을 위한 VM생성부터 네트워크 환경 설정, Kuberentes 설치, 레지스트리 구성, Kubeflow 이미지 설치 등 Kubeflow 구성을 직접 수행해야 합니다.

      SCP Kubeflow를 사용하면 SCP 콘솔에서 간단하게 고객의 워커노드 자원에 Kubeflow를 프로비저닝할 수 있으며, SCP에서 검증한 Kubeflow버젼과 이를 기반으로 한 다양한 add-on Feature들이 포함된 소프트웨어를 사용할 수 있습니다.

      SCP는 Enterprise 레벨에서 활용할 수 있는 다양한 Machine Learning 기능들을 개발하여 추가하고 있습니다.
    • Logging & Audit

      Trail 은 어떤 경우에 사용하는 것인가요?

      Trail을 생성하여, 사용자가 직접 사용자의 오브젝트스토리지에 로그를 저장할 수 있는 기능입니다. Trail을 생성하실때에는 로그를 수집/저장하고자 하는 상품과 사용자계정을 선택하여 생성할 수 있습니다. 사용자 오브젝트스토리지에 저장된 로그는 별도의 저장기간 제한이 없습니다.
    • Cloud Monitoring

      Cloud Monitoring은 어떤 서비스를 제공하나요?

      Cloud Monitoring은 대시보드, 이벤트/알림, 로그 모니터링 등 리소스를 안정적으로 운영하기 위한 모니터링 서비스를 제공합니다.
      1) 대시 보드 생성/관리
      - 프로젝트 대시보드(기본) : 대상 자원, 운영상태, 성능 Top5, 이벤트 맵, 이벤트 리스트
      - 사용자 정의 대시 보드
      · 기본 위젯 : Title, 이벤트 목록, Overview(대상수, 유형별 이벤트 수 등), Top5(CPU 사용율, 메모리 사용율 등) 등
      · 사용자 정의 위젯 : 시계열 그래프, 비교차트, 인스턴스 맵 등
      2) 이벤트/알림 관리
      - 이벤트 지표 설정 : 이벤트 패턴, 스케줄링, 발생조건, 대상 정보 등
      - 사용자 정의 이벤트 제공 : 모니터링 대상, 위험도, 성능항목, 이벤트 룰 등
      3) 로그 모니터링
      - 프로젝트별 로그현황 : 누적로그 수, 일별 로그 발생량/누적율, 대상별 로그 발생량
      - 로그 수집/저장/검색
      - 검색된 로그데이터 결과 내보내기(CSV 파일)
      - 검색 키워드 이벤트 지표화 및 알림 통보
      4) 연계 상품
      - 성능 모니터링 : Virtual Server, Bare Metal, VM Auto Scaling, FileStorage, VPC, DB 9종, LoadBalance, K8s Engine.
      - 로그 모니터링 : Virtual Server, Bare Metal, K8s Engine, DB 9종
      ※ 성능/로그 모니터링의 연계 상품은 지속적으로 확대 예정입니다.
    • 요금

      요금은 어떻게 산정되나요?

      사용하시는 상품에 따라서 다양한 요금 산정 기준이 적용됩니다.
      약정기간동안 동일한 요금이 발생하는 상품, 이용한 시간만큼 요금이 발생하는 상품, 사용량만큼 과금 되는 상품, 건수만큼 요금이 발생하는 상품, 할당량(사용량)에 따라 할인이 적용되는 상품 등 다양한 과금 기준이 있습니다.
      요금표에 표시된 요금은 부가세 별도이며, 사용 전 상품별 과금 기준을 참고하시기 바랍니다.

      서버 사용료 과금 기준은 어떻게 되나요?

      무약정 또는 1년/3년 약정요금을 선택할 수 있으며, 사용 OS에 따라서 요금이 결정됩니다.
      무약정 선택시 사용한 시간에 따라 시간단위 요금이 적용되며, 서버를 OFF하는 경우에는 서버 사용 요금이 발생하지 않습니다.
      서버에 연결된 스토리지 등 서비스가 유지되는 자원의 경우 정상적으로 요금이 발생합니다.

      약정 요금 적용 기준은 어떻게 되나요?

      약정을 선택하면 해당 상품의 사용여부와 무관하게 월 과금됩니다.
      약정이 종료된 시점에는 해당 종료시점의 무약정 요금으로 자동 전환됩니다.
      약정 요금은 Compute 등 일부 상품에 한하여 제공됩니다.
    • 요금

      서비스 해지 시 위약금이 발생하나요?

      무약정 요금의 경우 별도의 해지 위약금 없이 언제든지 해지가 가능합니다.
      다만, 약정 요금의 경우는 잔여 약정기간 개월수에 해당하는 총 서비스료의 50%를 해지위약금으로 청구합니다.
      해지 위약금은 해지한 다음달 청구일에 해지일을 기준으로 일할계산된 사용료와 함께 청구됩니다.

      스토리지 과금 기준은 어떻게 되나요?

      Block Storage, File Storage는 해당 월의 할당된 용량(GB)에 대해서 과금하며, Object Storage와 Backup은 해당 월의 사용한 용량(GB)에 대해서 과금합니다.
      상품 별 상세 과금 기준을 참고하시기 바랍니다.

    • VM Auto-Scaling

      어떤 기준으로 자원 규모가 조정됩니까?

      VM Auto-Scaling의 실행 조건은 크게 2가지 기준으로 나눌수 있습니다. 첫번째는 자원 사용량 입니다. 사용자는 CPU/메모리 등의 임계치/평균값 등을 지정하여, 자원 생성/삭제의 기준을 정의할 수 있습니다. 두번째는 일정입니다. 사용자는 희망하는 스케쥴을 사전 등록하여, 해당 시점에 자원의 자동 생성/삭제를 설정할 수 있습니다.
    • Bare Metal Server

      Virtual Server와 다른 Bare Metal Server의 제약 조건은 무엇입니까?

      단독 자원을 제공하기 때문에 일부 제공 되지 않는 기능들이 있으며, 지속적으로 개선을 위해 검토 중 입니다.
      - Security Group 설정, Custom Image 생성, Init script 입력 등 기능 미제공
      - 한개의 프로젝트 내 Bare Metal Server는 최대 20개까지 생성 가능
    • Block Storage

      블록스토리지 볼륨 생성시, 최대 할당량은 얼마인가요 ?

      기본적으로 서버의 OS영역은 100GB를 제공하며, 이후 외장 블록 스토리지를 통해 160TB 까지 온라인 중에 할당 가능합니다.
    • File Storage

      파일스토리지 볼륨 생성시, 최대 할당량은 얼마인가요?

      파일스토리지의 볼륨을 생성 시, 최대 100TB 까지 할당할 수 있습니다.
    • Object Storage

      과금은 어떻게 되나요?

      버킷 내 사용량을 측정하여 일단위로 계산되어 과금됩니다.
    • Backup

      백업으로 제공하는 대상은 어떤게 있나요?

      VM 이미지 스냅샷, NAS 이미지 스냅샷, DB 백업(Online) 을 제공합니다.
    • DB Service

      DB Service는 무엇인가요?

      DB Service는 웹 기반 콘솔을 통해 관계형, 비정형 및 데이터 분석 관련 DBMS에 대해 데이터베이스 생성과 설정을 자동화된 구성과 환경으로 제공합니다. 중요 데이터의 안정적인 관리를 위해 VM 환경에서 동작하며, VPC 기반으로 단일 인스턴스나 이중화 구성이 가능하고, 백업/복구, Parameter 관리, Audit , Minor Patch, 모니터링 등 데이터베이스의 생명주기 내 업무를 자동화 기능으로 제공합니다.
    • Kafka

      Kafka를 시작하려면 어떻게 해야 하나요?

      사용자는 SCP Kubernetes Engine을 사용해야 합니다. 만약 그렇지 않은 경우 SCP Kubernetes Engine 사용신청을 통해 사용가능한 워크노드(일반 VM)가 신청합니다. SCP Kubernetes Engine에서 프로비저닝한 워커노드(일반 및 GPU VM)를 Kafka의 자원으로 이용하게 됩니다.
    • API Gateway

      API 등록 시, Endpoint와 연결이 되었는지 확인이 가능한가요?

      네 가능합니다.
      삼성SDS 클라우드 콘솔에 로그인하여 API 상품 화면으로 접속하시어, Service 생성 후 좌측에 [API 관리 > API 등록]에서 'Connection Test' 클릭 시, 확인 가능합니다.
    • Mail/SMS/Push

      API형태로 제공되는 Mail/SMS/Push 서비스 이용 방법이 어떻게 되나요?

      삼성SDS 클라우드 콘솔에 로그인 한 후, 우측 [모든 상품 → Application Service → Mail/SMS/Push]를 통해 상품신청을 합니다. 이 후, 신청된 각 상품별로 [상품목록 → Mail/SMS/Push] 클릭, [사용자 가이드] 조회를 통해 가이드내 API들을 직접 커스텀하여 사용 가능합니다.
    • Kubernetes Engine

      SCP Kubernetes Engine(SKE)를 사용해야 하는 이유는 무엇입니까?

      SCP Kubernetes Engine(SKE) 없이 컨테이너 환경을 구성하고 싶다면, 컨트롤플레인과 워커노드 구성을 위한 VM생성부터 Kuberentes 설치 등 Kubernetes 클러스터 구성을 직접 수행해야 합니다.

      SCP Kubernetes Engine(SKE)를 사용하면 SCP 콘솔에서 간단하게 워커 노드를 프로비저닝할 수 있으며, SCP가 보장하는 가용성의 컨트롤 플레인을 제공받을 수 있습니다. 이를 통해 Kubernetes를 실행하는 데 따른 운영 부담을 현저히 줄이고 컨테이너 환경 구성을 위한 노력을 애플리케이션 구축에 활용할 수 있습니다.

      SCP는 고가용성 보장을 위해 비정상적인 컨트롤 플레인 노드를 감지하고 교체하며 컨트롤 플레인에 대한 패치를 제공합니다. 또한 다양한 SCP의 상품들과 통합하여 애플리케이션에 확장성과 보안을 강화할 수 있습니다.
    • Kubernetes Apps

      Kubernetes Apps는 무엇인가요?

      Kubernetes Apps는 Web/WAS를 비롯한 App Infrastructure 등 다양한 컨테이너 이미지를 Kubernetes Engine으로 생성한 클러스터에 배포하는 기능을 제공하는 서비스입니다.
      다수의 현장 운영 경험을 기반으로 최적화 된 설정의 컨테이너 이미지를 제공하기 때문에 손쉬운 Apps 배포가 가능합니다.
    • VPC

      기존에 사설망(고객망)에서 사용하던 비공인 IP를 클라우드 환경에서도 그대로 사용할 수 있나요?

      고객망에서 사용하던 비공인IP 도 사용 가능합니다만, 마이그레이션 상황(순차적 이관 등)에 따라 기존 사용하던 비공인IP를 그대로 사용하지 못할 수도 있습니다. 그리고 기존 사설망(고객망)과 연동을 위해서는 Direct Connect 또는 VPN 상품을 통해 연결해야 합니다.
    • Security Group

      Security Group과 VPC의 방화벽의 차이점은 무엇인가요?

      Networking의 Security Group은 VM을 위한 방화벽입니다. Security Group에서 작성한 방화벽 Rule은 Virtual Server, Auto-Scaling, DB, Load Balancer 등에서 사용할 수 있습니다.
      VPC 상품 내 방화벽 탭에서 설정하는 룰은 VPC Gateway에서 동작합니다.
    • Load Balancer

      Load Balancer에서 생성할 수 있는 서비스는 어떤것이 있나요?

      L4 TCP, L7 HTTP 그리고 L7 HTTPS를 제공합니다.
      HTTP/HTTPS의 경우 URL 패턴에 따라 서비스를 전달할 서버그룹을 다르게 지정할 수 있습니다.
      HTTP와 HTTPS 선택 시 URL 처리 입력이 활성화 되고, HTTPS 입력 시 SSL인증서를 필수로 지정해야 합니다.
    • DNS

      이미 보유 중인 도메인은 등록할 수 없는가요?

      신규 도메인 신청만 가능하며, 기존 고객이 보유 중인 도메인 등록은 아직 안됩니다.
      ※ 신규 도메인은 1년 단위로 신청되며, 도메인등로 건당 사용료(자동연장) 별도 청구됩니다.
    • GSLB

      GSLB는 언제 사용하나요?

      GSLB는 DNS 기반으로 지역별 트래픽 증가시 트래픽을 분산하고 특정 서버 장애시 Fail over하여 서비스가 지속될 수 있도록 하는 상품으로, DC(Data Cente)r에 문제가 생길 경우에 다른 지역DC로 연결하여 시스템 가용성을 높여줍니다.
    • VPN

      호환되는 VPN 장비는 어떤 것이 있나요?

      시스코, 팔로알토, 체크포인트, 주니퍼 등이며 IPsec표준을 통해 연동합니다.
      VPN 표준을 준수했다면 연동 시 문제는 크게 없을 것입니다.
      연동 후 안정화 기간을 두고 점검하길 권고드립니다.
    • DDoS 대응

      WAF를 신청하면 DDoS 대응은 자동으로 신청되나요?

      DDoS 대응 상품은 WAF 상품을 이용하는 고객에 한하여 제공됩니다. (Basic 등급에 한함) 다만, DDoS 대응 상품을 이용하기 위해서는 WAF 신청 이후 별도로 신청을 해주셔야 합니다.
    • IDS

      IDS Agent 설치 및 삭제는 누가 하는 것인가요?

      VM에 IDS Agent 설치 및 삭제는 고객이 수행합니다. 서비스 신청 시 E-mail을 통해 관제센터에서 매뉴얼을 제공해 드립니다.
    • WAF

      탐지 모드에서 공격을 시도한 IP 차단이 되나요?

      DDoS 대응 상품은 WAF 상품을 이용하는 고객에 한하여 제공됩니다. (Basic 등급에 한함) 다만, DDoS 대응 상품을 이용하기 위해서는 WAF 신청 이후 별도로 신청을 해주셔야 합니다.
    • 웹 취약점 점검

      웹 취약점 점검 이후 재진단이 하고 싶습니다.

      웹 취약점 점검 완료 후 발견된 취약점에 대해 이행검수를 제공합니다. 취약점에 대해 검수 요청을 해주시면 수동 점검을 수행한 후 결과 Report를 제공해 드립니다. 이행검수 제공기간은 취약점 점검 완료 후 30일 내이며 총 2회까지 요청하실 수 있습니다.
    • 인증서 관리

      인증서 관리는 어떤 상품인가요?

      SSL/TLS 인증서를 등록하여 SDS Cloud를 통해 제공되는 웹사이트/어플리케이션을 보호할 수 있습니다. 또한 인증서 조회, 유효기간 만료 알림 설정 등 통합인증서 관리 기능을 제공합니다.
    • Kubeflow

      SCP Kubeflow를 사용해야 하는 이유는 무엇입니까?

      SCP Kubeflow 없이 ML 환경을 구성하고 싶다면, Kuberntes 컨트롤플레인과 워커노드 구성을 위한 VM생성부터 네트워크 환경 설정, Kuberentes 설치, 레지스트리 구성, Kubeflow 이미지 설치 등 Kubeflow 구성을 직접 수행해야 합니다.

      SCP Kubeflow를 사용하면 SCP 콘솔에서 간단하게 고객의 워커노드 자원에 Kubeflow를 프로비저닝할 수 있으며, SCP에서 검증한 Kubeflow버젼과 이를 기반으로 한 다양한 add-on Feature들이 포함된 소프트웨어를 사용할 수 있습니다.

      SCP는 Enterprise 레벨에서 활용할 수 있는 다양한 Machine Learning 기능들을 개발하여 추가하고 있습니다.
    • Logging & Audit

      Trail 은 어떤 경우에 사용하는 것인가요?

      Trail을 생성하여, 사용자가 직접 사용자의 오브젝트스토리지에 로그를 저장할 수 있는 기능입니다. Trail을 생성하실때에는 로그를 수집/저장하고자 하는 상품과 사용자계정을 선택하여 생성할 수 있습니다. 사용자 오브젝트스토리지에 저장된 로그는 별도의 저장기간 제한이 없습니다.
    • Cloud Monitoring

      Cloud Monitoring은 어떤 서비스를 제공하나요?

      Cloud Monitoring은 대시보드, 이벤트/알림, 로그 모니터링 등 리소스를 안정적으로 운영하기 위한 모니터링 서비스를 제공합니다.
      1) 대시 보드 생성/관리
      - 프로젝트 대시보드(기본) : 대상 자원, 운영상태, 성능 Top5, 이벤트 맵, 이벤트 리스트
      - 사용자 정의 대시 보드
      · 기본 위젯 : Title, 이벤트 목록, Overview(대상수, 유형별 이벤트 수 등), Top5(CPU 사용율, 메모리 사용율 등) 등
      · 사용자 정의 위젯 : 시계열 그래프, 비교차트, 인스턴스 맵 등
      2) 이벤트/알림 관리
      - 이벤트 지표 설정 : 이벤트 패턴, 스케줄링, 발생조건, 대상 정보 등
      - 사용자 정의 이벤트 제공 : 모니터링 대상, 위험도, 성능항목, 이벤트 룰 등
      3) 로그 모니터링
      - 프로젝트별 로그현황 : 누적로그 수, 일별 로그 발생량/누적율, 대상별 로그 발생량
      - 로그 수집/저장/검색
      - 검색된 로그데이터 결과 내보내기(CSV 파일)
      - 검색 키워드 이벤트 지표화 및 알림 통보
      4) 연계 상품
      - 성능 모니터링 : Virtual Server, Bare Metal, VM Auto Scaling, FileStorage, VPC, DB 9종, LoadBalance, K8s Engine.
      - 로그 모니터링 : Virtual Server, Bare Metal, K8s Engine, DB 9종
      ※ 성능/로그 모니터링의 연계 상품은 지속적으로 확대 예정입니다.

  • 요금(10)

    • 요금

      요금은 어떻게 산정되나요?

      사용하시는 상품에 따라서 다양한 요금 산정 기준이 적용됩니다.
      약정기간동안 동일한 요금이 발생하는 상품, 이용한 시간만큼 요금이 발생하는 상품, 사용량만큼 과금 되는 상품, 건수만큼 요금이 발생하는 상품, 할당량(사용량)에 따라 할인이 적용되는 상품 등 다양한 과금 기준이 있습니다.
      요금표에 표시된 요금은 부가세 별도이며, 사용 전 상품별 과금 기준을 참고하시기 바랍니다.
    • 요금

      서비스 미터링 기준 시간대는 무엇인가요?

      현재 서비스 미터링 시간대는 '한국(UTC+9)'입니다.
    • 요금

      서버 사용료 과금 기준은 어떻게 되나요?

      무약정 또는 1년/3년 약정요금을 선택할 수 있으며, 사용 OS에 따라서 요금이 결정됩니다.
      무약정 선택시 사용한 시간에 따라 시간단위 요금이 적용되며, 서버를 OFF하는 경우에는 서버 사용 요금이 발생하지 않습니다.
      서버에 연결된 스토리지 등 서비스가 유지되는 자원의 경우 정상적으로 요금이 발생합니다.
    • 요금

      약정 요금 적용 기준은 어떻게 되나요?

      약정을 선택하면 해당 상품의 사용여부와 무관하게 월 과금됩니다.
      약정이 종료된 시점에는 해당 종료시점의 무약정 요금으로 자동 전환됩니다.
      약정 요금은 Compute 등 일부 상품에 한하여 제공됩니다.
    • 요금

      서비스 해지 시 위약금이 발생하나요?

      무약정 요금의 경우 별도의 해지 위약금 없이 언제든지 해지가 가능합니다.
      다만, 약정 요금의 경우는 잔여 약정기간 개월수에 해당하는 총 서비스료의 50%를 해지위약금으로 청구합니다.
      해지 위약금은 해지한 다음달 청구일에 해지일을 기준으로 일할계산된 사용료와 함께 청구됩니다.
    • 요금

      스토리지 과금 기준은 어떻게 되나요?

      Block Storage, File Storage는 해당 월의 할당된 용량(GB)에 대해서 과금하며, Object Storage와 Backup은 해당 월의 사용한 용량(GB)에 대해서 과금합니다.
      상품 별 상세 과금 기준을 참고하시기 바랍니다.
    • 요금

      사용 금액은 언제 확인할 수 있나요?

      사용요금은 삼성SDS 클라우드 콘솔 내 [관리 > 통합 이용관리] 메뉴에서 확인가능하며, 이용내역에서는 일별 요금(예상금액)을 확인 하실수 있고, 청구 내역은 월초에 전월 요금이 확정된 이후에 확인 가능합니다.
      이용내역은 예상요금 이므로 청구내역과 상이할 수 있습니다.
    • 요금

      예상 요금 조회는 어떻게 할 수 있나요?

      삼성SDS 클라우드 서비스 포탈 내 요금 계산기를 이용하여 상품별 예상 요금을 조회할 수 있습니다.
    • 요금

      용량을 더 많이 사용할수록 할인이 있나요?

      용량 할인 적용 유무는 상품별로 기준이 다릅니다.
      예를 들어 스토리지(Block, File, Object, Backup)의 경우 월 할당량(또는 사용량)에 따라 서비스 요금이 계층화 되어있습니다.
      즉, 더 많이 사용 할수록 gb당 요금은 줄어듭니다. 상품별 상세 과금 기준을 참고하시기 바랍니다.

      1. 로그인 페이지 하단의 ID/비밀번호 찾기를 클릭합니다.
      2. ID 찾기에서는 국가번호를 선택하고 모바일 번호를 입력 후 인증 버튼을 클릭합니다.

      회원 가입은 어떻게 하나요?

      콘솔을 이용하기 위해서는 먼저 회원가입을 해야합니다.
      1. 로그인 페이지에서 의 [회원가입]을 클릭하세요.
      2. 회원가입 페이지에서 '약관동의' 항목에 대해서 동의에 체크하세요.
      3. Knox 사용자(삼성관계사 사내 계정 사용)일 경우 Yes를 선택해 주세요.
      4. ID(이메일)필드에 메일을 입력 후 중복을 체크하세요. Knox 사용자는 메일 입력 후 유효성을 검증하세요.
      5. 이름/성/회사/부서를 입력하세요. Knox 사용자는 시스템에서 자동으로 정보를 가져옵니다.
      6. 비밀번호는 영문과 숫자, 특수문자(!@#$%&*^)를 조합하여 8~20 문자로 입력 가능합니다.
      7. 콘솔에서 사용할 언어를 선택합니다.
      8. 2단계 인증을 선택합니다. "사용안함"을 선택해도 SMS 혹은 이메일을 통해 2단계 인증을 통과해야 로그인 할 수 있습니다.
      9. 접근제어IP를 설정합니다. Yes로 설정하고 IP를 등록한 경우, 등록된 IP를 통해서만 접근이 가능합니다. 신중히 등록해주세요.
      10. 확인를 클릭하여 회원 가입 절차를 완료합니다.

      로그인 2차 인증은 무엇인가요?

      ID/Password를 입력하면 가입하신 이메일 주소 또는 휴대폰 번호로 One Time Password가 발급됩니다.
